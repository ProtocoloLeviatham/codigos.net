// 0x01 RECURSOS EN LÍNEA: Herramientas Kali Linux //

[ RECONOCIMIENTO ] Nmap: Escaneo Completo de Servicios y OS

Nmap es el estándar para el descubrimiento de redes y auditorías de seguridad. El comando `-A` habilita la detección del sistema operativo y las versiones de los servicios, `-T4` acelera el escaneo. Siempre con permiso.


# Escaneo agresivo (OS y Versión) de un rango de IPs
nmap -A -T4 192.168.1.0/24

Este escaneo puede generar mucho tráfico de red, úsalo con precaución y solo en entornos autorizados.

[ OSINT ] theHarvester: Recolección de Correos y Subdominios

theHarvester es una herramienta de OSINT que busca direcciones de correo electrónico, subdominios, nombres de host, y más de fuentes públicas como Google, Bing, LinkedIn y Shodan. Es un excelente punto de partida para la fase de reconocimiento.


# Buscar correos y subdominios para 'ejemplo.com' desde Google y Baidu
theharvester -d ejemplo.com -l 500 -b google,baidu

La opción `-l` limita el número de resultados y `-b` especifica las fuentes. Siempre respeta las políticas de uso de cada motor de búsqueda.

[ ANÁLISIS WEB ] Nikto: Escáner de Vulnerabilidades Web

Nikto es un escáner de vulnerabilidades de servidores web de código abierto. Realiza pruebas completas contra servidores web para múltiples elementos, incluyendo más de 6700 archivos/programas potencialmente peligrosos, verificaciones de versiones obsoletas, y problemas específicos de servidores.


# Escaneo básico contra un objetivo web
nikto -h http://target.com

Nikto es bastante ruidoso y puede ser detectado fácilmente por un WAF (Web Application Firewall). Es más adecuado para las primeras etapas de un pentest interno.

// 0x02 SEGURIDAD DEFENSIVA: Hardening y Prevención //

[ HARDENING ] Deshabilitar SSH Root Login y Auth por Contraseña

Para securizar un servidor SSH, es fundamental deshabilitar el acceso directo como `root` y la autenticación por contraseña, forzando el uso de claves SSH. Edita el archivo `/etc/ssh/sshd_config`.


# Archivo: /etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no

Después de editar, reinicia el servicio SSH: `sudo systemctl restart sshd`. Asegúrate de tener una clave SSH configurada y funcionando antes de deshabilitar la autenticación por contraseña.

[ FIREWALL ] Reglas Básicas con UFW (Uncomplicated Firewall)

UFW es una interfaz para `iptables` que simplifica la configuración del firewall en Linux. Aquí, reglas para permitir SSH (puerto 22) y denegar todo lo demás por defecto.


# Denegar todas las conexiones entrantes por defecto
sudo ufw default deny incoming
# Permitir todas las conexiones salientes por defecto
sudo ufw default allow outgoing

# Permitir SSH (puerto 22)
sudo ufw allow ssh

# Habilitar el firewall
sudo ufw enable

Siempre verifica el estado con `sudo ufw status` y ten precaución al configurar firewalls, podrías bloquear tu propio acceso.

// 0x03 ARCHIVO HISTÓRICO: Explotación y Conocimiento //

[ SQLI ] SQLMap: Inyección SQL Automatizada

SQLMap es una herramienta de código abierto que automatiza el proceso de detección y explotación de vulnerabilidades de inyección SQL. Soporta una amplia gama de sistemas de bases de datos y técnicas de inyección.


# Escaneo básico de un parámetro vulnerable
sqlmap -u "http://target.com/vuln_page.php?id=1" --dbs

Este comando intenta listar las bases de datos disponibles si el parámetro `id` es vulnerable. **¡Extremadamente potente, solo usar en entornos autorizados!**

[ FUERZA BRUTA ] Hydra: Ataques de Contraseñas a Servicios

Hydra es una herramienta de ataque de fuerza bruta rápida y flexible, capaz de atacar más de 50 protocolos incluyendo FTP, SSH, Telnet, HTTP, SMB, y muchos más. Es esencial para pruebas de robustez de contraseñas.


# Fuerza bruta a SSH con lista de usuarios y contraseñas
hydra -L users.txt -P passwords.txt ssh://192.168.1.100

Los archivos `users.txt` y `passwords.txt` deben contener listas de credenciales para probar. Utiliza esto solo con consentimiento explícito.

[ XSS ] Cross-Site Scripting (Reflejado)

XSS permite a un atacante inyectar scripts maliciosos en páginas web que luego son ejecutados por otros usuarios. Un payload simple para XSS reflejado:


<script>alert('XSS Exitoso por KaliHunter32');</script>

Este payload causa que se muestre una ventana emergente en el navegador del usuario. Un atacante real intentaría robar cookies o redirigir a sitios maliciosos.
